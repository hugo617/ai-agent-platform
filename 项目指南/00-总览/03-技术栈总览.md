## 🎯 一句话定位

**多租户 AI 智能体 SaaS 平台**,可作为新 SaaS 产品的脚手架。

后端 Python 全栈(FastAPI)、前端 React 全栈(Vite)、PostgreSQL 16 + pgvector、双轨认证(本地 bcrypt + Logto OIDC)。

## 🔐 认证(双轨)

| 方式 | 算法 | 用途 |
|---|---|---|
| 本地密码登录 | HS256 | 内建账号密码,bcrypt 哈希 |
| Logto OIDC | RS256 | 第三方 IdP,cryptography 验签 |
| 开发 token | RS256 | 本地/CI 调试捷径 |

**关键**:三种 token 都走同一条 `get_current_user` 验证管线,后端业务代码无感知。

## 🔁 二开脚手架:什么能换,什么不能换

如果你拿这个项目作为新 SaaS 产品的脚手架,以下是替换指南。

### ✅ 容易换(独立模块,改了不扩散)

| 模块 | 怎么换 |
|---|---|
| **LLM 提供商** | 改 `app/core/config.py` 的 LLM_* 配置 + `langchain-openai` 兼容协议可接 DeepSeek/Qwen/Kimi 等 |
| **embedding 提供商** | 改 `EMBEDDING_*` 配置 + `app/services/embedding_service.py`(注意 `check_embedding_ctx_length=False` 对 ollama 必须) |
| **OIDC 提供商** | Logto → Auth0/Authentik/Keycloak,改 `auth-logto` 模块的 JWKS_URL/ISSUER |
| **对象存储** | S3 → OSS/COS/MinIO,实现 `StorageBackend` 接口(参考 `LocalStorage` / `AmazonS3Storage`) |
| **UI 主题** | tailwind 配置 + shadcn 组件源码,自由改 |
| **业务模块** | agents/customers/groups 等是范例,可删可改 |

### ⚠️ 谨慎换(改了会牵动多处)

| 模块 | 风险点 |
|---|---|
| **数据库 PG → MySQL** | pgvector 无对应;部分索引语法差异;casbin 适配器要换 |
| **认证体系** | 改算法/管线会牵动 60+ 处 `require_permission` caller + graph.py 工具内 check |
| **RBAC 模型** | pycasbin → 自研要重写 PermissionService + DataScopeService |

### 🚫 不要换(项目骨架)

| 模块 | 为什么 |
|---|---|
| **四层分层**(Controller→Service→Repository→Model) | 是项目最核心的依赖方向约定,反向 = 技术债 |
| **多租户隔离**(tenant_id + Repository 自动过滤) | 是 SaaS 的合规底线 |
| **WIP=1 + 完成绑定证据**(harness 工程规则) | 防 agent 越界/半成品/假完成 |

> 详见 [04-二开脚手架/02-新增后端模块](../04-二开脚手架/02-新增后端模块.md)。
