## 🔧 后端任务

### 改/加数据库字段

**看**:[03-数据库与ORM](../02-后端架构/03-数据库与ORM.md)

1. 改 Model(`app/models/xxx.py`),加/改列定义。
2. `alembic revision --autogenerate -m "说明"`。
3. **检查**生成的迁移文件 `upgrade()` 是否正确。
4. `alembic upgrade head`。
5. 改 Schema(`app/schemas/xxx.py`)让接口能用到新字段。
6. 改前端 `types.ts` 保持一致。

---

### 加一个新接口(已有模块)

**看**:[01-分层架构](../02-后端架构/01-分层架构与依赖方向.md)

1. Schema(`schemas/`)加请求/响应类型。
2. Service(`services/`)加方法(开头 `permission_service.require`)。
3. API(`api/v1/xxx.py`)加路由(声明 `require_permission`)。
4. 如需新权限 → 改 `permission_service.seed_tenant_defaults`。
5. 前端 `endpoints.ts` + `queries.ts` 加对应函数/hook。

---

### 加全新业务模块(后端)

**看**:[04-二开/02-新增后端模块](../04-二开脚手架/02-新增后端模块.md)(完整 8 步)

Model → Repo → Schema → Service → API → 注册 → 迁移 → 权限 seed。照着 products 范例走。

---

### 改默认角色权限

**看**:[06-权限模型RBAC](../02-后端架构/06-权限模型RBAC.md)

- 改 `app/services/permission_service.py` 的 `seed_tenant_defaults` 里 owner/admin/member
  的权限列表。
- ⚠️ 只对**新建**的租户生效。已存在的租户要手动调 `add_policy` 补。

---

### 加 AI 工具

**看**:[07-Agent与LLM集成](../02-后端架构/07-Agent与LLM集成.md)

1. 在 `app/agents/graph.py` 的 `_build_tenant_tools` 里加 `@tool` 函数。
2. 工具内部**第一行**加 `permission_service.check(...)`。
3. 用 `AgentRepository(db)` 等查数据(自动带租户隔离)。

---

### 改配置项

**看**:[02-配置与环境](../02-后端架构/02-配置与环境.md)

1. `app/core/config.py` 的 `Settings` 加字段(带默认值)。
2. `.env.example` 加一行。
3. 代码用 `settings.xxx` 读。
4. 改完 `.env` 要**重启**后端。

## 🚀 运维/启动任务

### 首次启动

**看**:[01-快速开始](../01-快速开始/02-启动项目.md)

`docker-compose up -d` → `alembic upgrade head` → `uvicorn ...` →
`python scripts/init_admin.py` → `cd frontend && npm run dev`

---

### 重置开发数据库

```bash
docker-compose down -v          # 删数据卷
docker-compose up -d            # 重启空库
alembic upgrade head            # 重建表
python scripts/init_admin.py    # 重建管理员
```

> ⚠️ 这会**清空所有数据**,只在开发环境用。

## 🐛 排查任务

### 403 Forbidden(权限不够)

**看**:[06-RBAC](../02-后端架构/06-权限模型RBAC.md)

- 该用户的角色在 casbin 里有没有对应策略?
- 新模块是否忘了在 `seed_tenant_defaults` 加权限?

### 401 Unauthorized(token 问题)

**看**:[05-认证](../02-后端架构/05-认证体系.md)

- token 过期?→ 重新登录。
- 账号被禁/移出租户?→ 会话重校验失败。
- 生产环境 JWT_SECRET 没改?→ 启动会报错。

### 跨租户数据泄漏/看不到数据

**看**:[04-多租户隔离](../02-后端架构/04-多租户隔离.md)

- 查询是否用了 `TenantScopedRepository`?
- User 类查询是否用了 join 关联表的方式?
- 创建时是否填了 `tenant_id`?

### AI 对话报错

**看**:[07-Agent](../02-后端架构/07-Agent与LLM集成.md)

- `OPENAI_API_KEY` 配了吗?
- agent 是否属于当前租户?
- 有 `conversations:chat` 权限吗?
