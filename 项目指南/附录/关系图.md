## 四、认证时序图(token 全旅程)

```mermaid
sequenceDiagram
    participant U as 浏览器
    participant B as 后端
    participant DB as 数据库

    Note over U,B: 登录(拿 token)
    U->>B: POST /auth/login (用户名+密码)
    B->>B: bcrypt 校验
    B->>DB: 写 UserSession
    B->>U: access_token (HS256)

    Note over U,B: 后续请求(带 token)
    U->>B: GET /api/v1/users (Bearer token)
    B->>B: decode_token 验签
    B->>B: 解析 tenant_id
    B->>DB: 重校验:账号有效?仍是成员?会话活跃?
    B->>DB: 查数据(WHERE tenant_id)
    B-->>U: 200 + 数据

    Note over U,B: 登出(token 失效)
    U->>B: POST /auth/logout (Bearer token)
    B->>DB: UserSession.is_active=false
    Note over B: 下次该 token 请求 → 重校验失败 → 401
```

## 五、多租户隔离机制

```mermaid
graph TB
    REQ[请求带 token] --> DEP[get_current_user 解析 tenant_id]
```

## 六、权限校验链

```mermaid
graph LR
    REQ[请求] --> C1{"Controller 层<br/>require_permission"}
    C1 -->|拒绝| F1[403]
    C1 -->|通过| SVC[Service 执行]
    SVC --> AI{是 AI 工具?}
    AI -->|是| C2{"工具内部<br/>permission_service.check"}
    C2 -->|拒绝| DENY[返回 ERROR]
    C2 -->|通过| DO[执行工具]
    AI -->|否| DO2[正常业务]
```

**双重校验**:Controller 声明式 + AI 工具内部再查。改权限两处都要想到。

## 七、前端数据流闭环

```mermaid
graph LR
    PAGE[页面] -->|读| Q["useQuery<br/>useProducts"]
    PAGE -->|写| M["useMutation<br/>useCreateProduct"]
    M -->|onSuccess| INV["invalidateQueries"]
    INV -->|失效缓存| Q
    Q -->|自动重查| EP[endpoints]
    EP --> CLI[client axios]
    CLI -->|401| EVT[发 AUTH_EXPIRED 事件]
    EVT --> AUTH[AuthProvider signOut]
    AUTH --> REDIR[跳 /login]
```
