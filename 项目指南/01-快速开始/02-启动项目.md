# 02 - 启动项目

📍 相关文档:[01-环境准备](01-环境准备.md) · [03-10分钟体验](03-10分钟体验.md)

> 这一篇手把手把项目跑起来。假设你已经完成了 [环境准备](01-环境准备.md)。
> 我们会启动后端、初始化数据库、创建管理员、启动前端,最后登录验证。

---

## 启动顺序一览

```
数据库(已在环境准备时启动)
     ↓
① 初始化数据库表结构(Alembic 迁移)
     ↓
② 启动后端(FastAPI)
     ↓
③ 创建管理员账号(只需首次)
     ↓
④ 启动前端(React)
     ↓
⑤ 打开浏览器登录 🎉
```

---

## ① 初始化数据库表结构

数据库虽然跑起来了,但里面是空的(没有表)。用 Alembic 建表:

```bash
# 确保还在项目根目录,且 .venv 已激活
alembic upgrade head
```

**这做了什么?** 执行所有数据库迁移,把表结构建好。包括 users、tenants、agents、
roles、permissions、organizations、sessions、logs 等等。

**验证成功**:输出里看到一系列 `Running upgrade ... -> ...`,最后没有报错。

> 💡 **什么是数据库迁移?** 类似给数据库表结构做「版本管理」。每次改表结构(加字段、
> 建新表)都生成一个迁移文件,`alembic upgrade head` 就是把所有迁移按顺序执行一遍。
> 详见 [03-数据库与ORM](../02-后端架构/03-数据库与ORM.md)。

> 🔧 **报错「无法连接数据库」?** 检查:
> - `docker-compose ps` 看 PostgreSQL 是否在跑
> - `.env` 里 `DATABASE_URL` 的端口(5433)、用户名、密码是否和 docker-compose 一致

## ② 启动后端

```bash
uvicorn app.main:app --reload
```

- `app.main:app` = `app/main.py` 文件里的 `app` 变量(后端入口)
- `--reload` = 代码改了自动重启(开发时方便)

**验证成功**:
1. 终端输出类似 `Uvicorn running on http://127.0.0.1:8000`
2. 浏览器打开 **http://localhost:8000/docs** —— 看到 FastAPI 自动生成的接口文档
   (一堆接口列表),说明后端起来了!
3. 打开 **http://localhost:8000/health** —— 返回 `{"status":"ok",...}`

> ⚠️ **这个终端窗口要保持开着!** 后端在跑。后面启动前端要**新开一个终端窗口**。

---

## ③ 创建管理员账号(首次)

数据库是空的,没有账号,登录不了。创建第一个管理员:

```bash
# 新开一个终端窗口,cd 到项目根目录,激活虚拟环境
source .venv/bin/activate

python scripts/init_admin.py
```

**默认创建**:
- 用户名:`admin`
- 邮箱:`admin@example.com`
- 密码:`Admin@123456`

**验证成功**:输出提示创建了租户 "Platform"、用户 admin、关联为 owner。

**想自定义?** 用环境变量覆盖:
```bash
ADMIN_USERNAME=myname ADMIN_EMAIL=me@xx.com ADMIN_PASSWORD=MyPass123 python scripts/init_admin.py
```

> 💡 这个脚本是**幂等**的——重复运行不会出错,已存在就跳过。

---

## ④ 启动前端

```bash
# 新开第三个终端窗口
cd frontend
npm run dev
```

**验证成功**:终端输出类似 `Local: http://localhost:3000/`。

> 💡 **为什么前端能直接调后端?** Vite 开发服务器配了代理,把 `/api`、`/dev`、
> `/oidc`、`/health` 开头的请求自动转发到后端的 8000 端口。所以前端用相对路径
> `/api/v1/...` 就能连上后端。配置在 `frontend/vite.config.ts`。

## ⑤ 打开浏览器登录

访问 **http://localhost:3000**,会跳到登录页。有**三种登录方式**任选:

### 方式 A:账号密码(推荐,刚创建的)

直接用刚创建的管理员:
- 用户名:`admin`(或邮箱 `admin@example.com`)
- 密码:`Admin@123456`

点登录 → 进入后台。🎉

### 方式 B:🚀 一键开发登录(最省事)

登录页有个「一键开发登录」按钮,**不用任何账号**,点一下就进去了。

**原理**:它调后端的 `/dev/bootstrap`(自动建一个开发租户+用户)和 `/dev/token`
(发一个开发 token)。**仅在 `APP_ENV=development` 下可用**,生产环境会 404。

> 适合快速看效果,不想创建账号时用。

### 方式 C:粘贴 Token

如果你已经有 token(比如用 curl 调 `/api/v1/auth/login` 拿到的),可以直接粘贴进去登录。
适合调试。

## 登录成功的标志

进入后台后,你应该看到:
- **左侧菜单**:Dashboard、Agents、Users、Roles、Permissions
- **右上角**:你的头像/用户名,有个登出按钮
- **页面能正常切换**,不报错

如果看到这些,恭喜!🎉 项目已经完全跑起来了。

---

## 现在开着的终端窗口

到这里你应该有 **3 个终端**在跑:
1. 后端:`uvicorn ...`(端口 8000)
2. 前端:`npm run dev`(端口 3000)
3. (数据库在 Docker 里跑,不需要专门开终端)

想停掉项目时,在对应终端按 `Ctrl + C`。数据库用 `docker-compose down`。

---

## 常见问题排查

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| 后端启动报「JWT_SECRET」错 | `.env` 里 `APP_ENV` 不是 development 且密钥没改 | 改 `JWT_SECRET` 或确保 `APP_ENV=development` |
| 登录提示「连接后端失败」 | 后端没启动 / 端口不对 | 确认 `http://localhost:8000/health` 能访问 |
| 前端报「代理错误」 | 后端没起 | 先启动后端 |
| 创建管理员报错 | 数据库没建表 | 先跑 `alembic upgrade head` |
| AI 对话报错 | `OPENAI_API_KEY` 没改 | 在 `.env` 填真实 key,重启后端 |
