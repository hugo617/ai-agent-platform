# 01 - 环境准备

📍 相关文档:[02-启动项目](02-启动项目.md) · [03-10分钟体验](03-10分钟体验.md)

> 这一篇帮你把开发环境准备好。装好依赖、配好配置文件,下一步就能跑起来了。
> 跟着步骤来,每步都有「成功标志」帮你确认。

---

## 你需要先装好这些(前置依赖)

在开始之前,请确认电脑上已经装了下面这些工具。打开终端(命令行)逐个验证:

| 工具 | 验证命令 | 没装怎么办 |
|------|---------|-----------|
| **Python 3.11+** | `python3 --version` | 去 python.org 下载,或用 `brew install python`(Mac) |
| **Node.js 18+** | `node --version` | 去 nodejs.org 下载,推荐 LTS 版 |
| **npm**(随 Node 装) | `npm --version` | 装 Node 就有了 |
| **Docker** | `docker --version` | 去 docker.com 下载 Docker Desktop |
| **Git** | `git --version` | 一般已装;没有去 git-scm.com |

> 💡 **为什么需要 Docker?** 项目用 PostgreSQL 数据库,用 Docker 一键拉起来最省事。
> 如果你本地已经装了 Postgres,也可以不用 Docker(但端口要对上,见下文)。

---

## 第 1 步:克隆代码(如果还没有)

如果你是从 GitHub 拿的代码:

```bash
git clone <仓库地址>
cd ai-agent-platform
```

如果代码已经在本地,直接 `cd` 进项目根目录就行。

---

## 第 2 步:准备配置文件 `.env`

项目根目录有个 `.env.example`(模板)。你需要复制一份成 `.env`(真正生效的配置):

```bash
cp .env.example .env
```

然后用编辑器打开 `.env`。下面**逐项解释**每个配置的含义,告诉你哪些要改:

### 应用相关
```ini
APP_NAME=ai-agent-platform        # 应用名,随便
APP_ENV=development               # 环境:development/testing/production
APP_DEBUG=true                    # 是否开调试模式(开发时 true)
API_V1_PREFIX=/api/v1             # 接口前缀,一般不用动
CORS_ORIGINS=["http://localhost:3000"]  # 允许的前端地址
```

### 数据库
```ini
POSTGRES_USER=aap                 # 数据库用户名
POSTGRES_PASSWORD=aap_secret      # 数据库密码
POSTGRES_DB=aap                   # 数据库名
POSTGRES_HOST=localhost
POSTGRES_PORT=5433                # ⚠️ 故意用 5433 避开本机默认的 5432
DATABASE_URL=postgresql+psycopg://aap:aap_secret@localhost:5433/aap
```
> 💡 **端口 5433 是故意的**:防止和你电脑上已有的 PostgreSQL(默认 5432)冲突。
> 跟着 docker-compose 走就行。

### 认证相关
```ini
LOGTO_ENDPOINT=http://localhost:3001       # Logto 服务地址(可选)
LOGTO_ISSUER=http://localhost:3001/oidc    # OIDC 签发者
LOGTO_AUDIENCE=http://localhost:8000/api   # token 受众
LOGTO_ADMIN_SUBJECT=admin-user

# ⚠️ 本地密码登录用的密钥,生产环境必须改!
JWT_SECRET=change-me-in-production
SALT_ROUNDS=12                    # bcrypt 强度,12 够用
ACCESS_TOKEN_TTL_MINUTES=60       # token 有效期 60 分钟
SESSION_TTL_HOURS=168             # 会话有效期 7 天

APP_URL=http://localhost:3000     # 前端地址(发邮件用)
```

> ⚠️ **重要安全提示**:`JWT_SECRET=change-me-in-production` 是默认占位值。
> - **本地开发**:保持不变没关系。
> - **生产环境**:**必须**改成一段随机长字符串,否则代码启动时会直接报错拒绝运行
>   (这是项目的保护机制,见 `app/core/config.py` 的 `_jwt_secret_not_default`)。

### AI 相关
```ini
OPENAI_API_KEY=sk-replace-me      # ⚠️ 要用 AI 对话功能必须改成你的真实 key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini          # 用哪个模型
```
> 💡 **没有 OpenAI key 也能跑**:其他功能(登录、用户管理)都正常,只是 AI 对话会失败。
> 如果用国内代理,把 `OPENAI_BASE_URL` 改成代理地址即可。

### 最低限度要改什么?

如果只想先把项目跑起来看看:
- **可以都不改**,保持默认值即可(本地开发模式)。
- 想用 AI 对话 → 改 `OPENAI_API_KEY`。
- 上线 → 必须改 `JWT_SECRET`。

---

## 第 3 步:启动数据库(Docker)

确保 Docker Desktop 已经打开运行,然后在项目根目录执行:

```bash
docker-compose up -d
```

这会启动两个服务:
- **PostgreSQL**(带 pgvector 向量扩展)—— 你的主数据库
- **Logto + Logto 数据库** —— 可选的登录服务(不用 Logto 登录可以忽略)

**验证成功**:
```bash
docker-compose ps
```
应该看到 `aap-postgres` 状态是 `running` 或 `healthy`。

> 🔧 **报「端口被占用」怎么办?**
> 说明 5433(或其他端口)已被占用。两种解法:
> 1. 改 `.env` 和 `docker-compose.yml` 里的端口(比如改成 5434)。
> 2. 停掉占用端口的其他程序。

---

## 第 4 步:安装后端依赖

```bash
python3 -m venv .venv              # 创建虚拟环境(隔离依赖)
source .venv/bin/activate          # 激活(macOS/Linux)
# Windows 用: .venv\Scripts\activate

pip install -r requirements-dev.txt  # 安装所有后端依赖
```

**验证成功**:终端提示符前面出现 `(.venv)`,且没有报错。

> 💡 **什么是虚拟环境?** 就是一个「独立的 Python 小房间」,装在这里的包不会污染系统。
> 项目根目录的 `.venv/` 就是它。详见 [术语表](../附录/术语表.md#虚拟环境venv)。

---

## 第 5 步:安装前端依赖

```bash
cd frontend
npm install
cd ..                              # 装完回到根目录
```

**验证成功**:`npm install` 跑完没报错,`frontend/node_modules/` 目录出现了。

> ⏱️ 这一步可能要几分钟(要下载几百个包),耐心等。
