约束图例
PK 主键
FK 外键
UK 唯一
NN 非空
CHECK 约束
IDX 索引
关联关系
多对多 · CASCADE
tenants ↔ users（经 user_tenants）
用户可加入多个租户，每成员关系带一个角色；任一端删除级联清除成员关系
一对多 · CASCADE
tenants → agents / conversations
租户删除时级联清除其下所有 Agent 定义与会话
一对多 · CASCADE
agents → conversations → messages
Agent 删除级联清会话；会话删除级联清消息
一对多 · CASCADE
tenants → roles / permissions
租户删除时级联清除其角色与权限定义（display 层，不影响 casbin_rule）
多对多 · CASCADE
roles ↔ permissions（经 role_permissions）
角色挂多个权限；任一端删除级联清除关联
软关联 · 无 FK
user_tenants.role → roles.code
成员角色名与 casbin 角色名一一对应，但不建 FK（保持 casbin 解耦）
自引用 · SET NULL
organizations.parent_id → organizations.id
组织树父子关系；父组织删除时子组织父级置空（不级联）
多对多 · CASCADE
users ↔ organizations（经 user_organizations）
用户可属多个组织，is_main 标记主组织
一对多 · CASCADE
users → user_sessions / user_login_methods
用户删除时级联清除其会话与登录方式记录
一对多 · SET NULL
users / tenants → system_logs
审计日志用 SET NULL：用户/租户删除后保留历史日志
软引用 · 无 FK
conversations.user_id → users.id
仅存 ID 不建外键：用户被删时保留会话历史用于审计
软引用 · 无 FK
messages.tenant_id → tenants.id
冗余租户 ID 加速隔离查询，Repository 层强制 WHERE tenant_id
独立表
casbin_rule（无 FK 关系）
由 casbin-sqlalchemy-adapter 自管，Alembic 已配置排除，不参与迁移
表详情按模块分组：租户核心 · 用户与权限 · 组织 · 会话与安全 · 审计
