## 一、一键复现(从空库到可演示)

### 前置条件
- Docker Desktop 已启动
- 已 `git clone` 本仓库并 `cd` 进项目根目录
- Python 3.13 + Node.js 已安装(或用仓库 `.venv`)

### 步骤

```bash
# 1. 起依赖服务:PostgreSQL(pgvector)+ Logto
docker-compose up -d

# 2. 跑全部数据库迁移(含最新结构)
alembic upgrade head

# 3. 创建平台超管(交互式输入用户名/密码,默认 admin / Admin@123456)
python scripts/init_admin.py

# 4. 灌入颐和堂演示数据
python scripts/seed_demo.py

# 5. 启动前端
cd frontend && npm install && npm run dev
```

打开 http://localhost:3000,开发环境登录页会**预填** `admin` / `Admin@123456`(生产环境不预填),点登录即可进入。

> 后端 API 若需单独启动:`uvicorn app.main:app --reload`(http://localhost:8000/docs 看 Swagger)。

## 二、`.env` 配置说明

复制 `.env.example` 为 `.env`,关键字段:

| 字段 | 作用 | 演示环境填什么 |
|------|------|--------------|
| `DATABASE_URL` | PostgreSQL 连接串 | 默认值即可(指向 docker 起的 PG) |
| `JWT_SECRET` | 本地 token 签名密钥 | 开发用默认;**生产必须改** |
| `FIELD_ENCRYPTION_KEY` | DB 中密钥加密用 | 开发用默认;**生产必须新生成** |
| `DEMO_LLM_API_KEY` | 演示用真实 LLM key(DeepSeek) | **有 key 才填**,留空则用占位符 |
| `DEMO_EMBEDDING_API_KEY` | 演示用真实 Embedding key(默认指向本地 Ollama) | 留空走占位符;本地 Ollama 已就绪时填任意非空值即可激活真实 RAG |
| `DEMO_LOGIN_USERNAME/PASSWORD` | 登录页预填值 | 默认 admin / Admin@123456 |

### 关于真实 Key(重要)

- **真 key 只进 `.env`**(已被 `.gitignore` 忽略),**绝不硬编码进代码**(会进 git 历史)
- `.env.example` 里只放**空值占位**,从不放真 key
- 填了真 key → `seed_demo.py` 会用真 key 灌入平台级 LlmConfig/EmbeddingConfig,**启动即可现场聊天 + 真实 RAG**
- 没填真 key → 用 `sk-demo-placeholder` 占位,文档仍会灌入(但 embed 失败标 `failed`),需事后在「设置」页填 key 才能真用 AI

## 三、数据全景(种子灌了什么)

`seed_demo.py` 一次灌入(幂等,可重复跑):

| 表 | 行数 | 关键内容 |
|----|------|---------|
| `tenants` | 3 | 朝阳理疗中心 / 海淀中医门诊 / 王府井理疗馆 |
| `users` | 8 | 7 店员/督导 + 超管来自 init_admin(共 8 账号) |
| `groups` | 2 | 颐和堂中医馆(朝阳+海淀)/ 独立养生馆(王府井) |
| `customers` + `customer_profiles` | 5 身份 / 8 档案 | 含跨店复用(张先生朝阳+海淀、刘女士朝阳+王府井);**含 active/vip/inactive/blacklist 4 态** |
| `agents` | 4 门店顾问 + 1 编排器 | 各店独立推理参数(temperature 0.2~0.9) |
| `agent_specialists` | 2 | 编排器挂 2 个专科顾问(理疗/中医) |
| `conversations` + `messages` | 5 段对话 | 中医问诊/理疗方案/艾灸注意/总部汇总(assistant 文本硬编码) |
| `documents` + `document_chunks` | 3 份 | 每店 1 份 RAG 文档(操作规范/禁忌/注意事项) |
| `llm_configs` | 2 | 平台级(deepseek-chat)+ 朝阳租户级覆盖(deepseek-reasoner) |
| `embedding_configs` | 1 | 平台级(bge-m3,本地 Ollama 提供;维度 1024) |
| `wallets` + `model_pricing` | 按需 | 门店钱包 + DeepSeek 单价 |
| `api_tokens` | 2 | 朝阳/海淀店主 AtoA 集成 token |
| `user_login_methods` | 1 | 陈馆长绑定手机号(演示多登录方式) |
| `system_logs` | 自动 | Service 调用自然产生的审计行 |

> 实时计数:种子跑完后终端会打印每步 created/exists;`--reset` 重跑会先清空再全量重建。

## 四、重置 / 清理

```bash
# 清空所有演示数据(白名单范围,绝不碰 super_admin 和非演示数据),然后全量重建
python scripts/seed_demo.py --reset

# 彻底清空数据库重来(连表结构)
docker-compose down -v          # 删 volume aap_pgdata
docker-compose up -d
alembic upgrade head
python scripts/init_admin.py
python scripts/seed_demo.py
```

`--reset` 的清理是**白名单精准删除**(按租户名/用户名/组织 code/客户手机号/Agent 名/角色 code 定位),不会误删你自己手动加的数据。

## 五、常见问题

### Q1:对话报 401 / 登录后无法对话?
- 检查是否用了有 `conversations:chat` 权限的账号(member 及以上都有)
- 检查后端是否启动(`uvicorn app.main:app --reload`)
- 开发环境可用「开发登录」按钮一键跳过密码(走 dev token)

### Q2:对话报「token 余额不足」?
- 该门店钱包余额耗尽。用 `admin` 登录 → `/billing/admin` 给对应门店充值

### Q3:AI 不回答 / 报错?
- 多半是没配真实 LLM key。`seed_demo` 默认用占位符
- 解决:`admin` 登录 → 「设置」(`/settings`)→ LLM 配置,填入真 DeepSeek key;或在 `.env` 设 `DEMO_LLM_API_KEY` 后重跑 seed

### Q4:知识库文档显示 `failed`?
- Embedding key 未配置。文档已入库但没生成向量
- 解决:`.env` 设 `DEMO_EMBEDDING_API_KEY`(OpenAI key)后 `python scripts/seed_demo.py --reset`

### Q5:如何换成别的行业(如美容、餐饮)?
- `scripts/seed_demo.py` 顶部的常量(`STORES` / `CUSTOMERS` / `AGENTS` / `DOCUMENTS` / `CONVERSATIONS`)就是全部业务内容
- 改这些常量即可换行业,架构(多租户/跨店/权限/计费)不变

### Q6:如何加第 4 家门店?
- 在 `STORES` 常量里加一项 `(店名, [(用户名, 显示名, 角色), ...])`
- 重跑 `python scripts/seed_demo.py`(增量,不碰已有数据)

## 六、验证数据是否灌成功

种子跑完后,关键检查:

```bash
# 后端启动后,用 admin 登录,或直接查库:
docker exec -it aap-postgres psql -U aap -d aap -c "
  SELECT 'tenants' t, count(*) FROM tenants
  UNION ALL SELECT 'users', count(*) FROM users
  UNION ALL SELECT 'agents', count(*) FROM agents WHERE is_deleted=false
  UNION ALL SELECT 'customers', count(*) FROM customers WHERE is_deleted=false
  UNION ALL SELECT 'documents', count(*) FROM documents WHERE is_deleted=false;
"
```

预期:tenants=3、users≥9、agents=5(含编排器)、customers=5(身份)、documents=3。

也可直接打开前端用各账号登录,对照 [02-演示账号清单](./02-演示账号清单.md) 验证能看到对应数据。
